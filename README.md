### 깃허브 커밋 규칙
| 타입         | 설명                             |
| ---------- | ------------------------------ |
| `feat`     | ✨ 새로운 기능 추가                    |
| `fix`      | 🐛 버그 수정                       |
| `refactor` | 🔨 리팩토링 (기능 변화 X, 구조 개선)       |
| `style`    | 💄 코드 포맷팅, 세미콜론, 들여쓰기 등        |
| `docs`     | 📝 문서 수정 (README 등)            |
